# Sprint 17 User Stories

## ODIAO reviewer

- As an ODIAO reviewer, I can see a queue of Open Data candidates needing assessment or review so I can prioritize publication decisions.
- As an ODIAO reviewer, I can inspect classification, quality, personal data, ownership, and publication value signals before approving publication.
- As an ODIAO reviewer, I cannot approve or publish candidates with unresolved blockers.

## Data owner

- As a data owner, I can register a governed asset as an Open Data candidate from Asset 360 so the publication process starts from a known asset.
- As a data owner, I can see whether ownership, steward assignment, classification, and quality readiness are complete enough for ODIAO review.

## Data steward

- As a data steward, I can update candidate metadata, frequency, format, and review notes so publication planning is clear.
- As a data steward, I can see which signals need work before publication.

## Auditor

- As an auditor, I can view Open Data candidates and their status so I can understand public-release readiness without editing the registry.
